>> and >>> bitwise operators

  • The >> operator preserves the sign bit, which means that the leftmost bit is replicated when shifting. This means that the result of a right shift can be negative if the original number was negative.
  • The >>> operator does not preserve the sign bit, which means that the leftmost bit is always set to 0 when shifting. This means that the result of a right shift is always a positive number.

Checked and Unchecked exceptions

  • Checked exceptions are exceptions that the code must handle or declare using the "throws" keyword
    Examples:

    • FileNotFoundException
    • IOException

  • Unchecked exceptionsthat the code does not have to handle or declare
    Examples:

    • NullPointerException
    • ArrayIndexOutOfBoundsException

throw and throws

  • "throw" is used to explicitly throw an exception. It is followed by an instance of the exception class that is being thrown.
  • "throws" is used to declare that a method might throw one or more exceptions. It is followed by the names of the exception classes that might be thrown.

Text and binary files

  • Text files are files that contain human-readable text. They can be opened and edited using a text editor like Notepad or WordPad
    Examples:

    1. .txt File
    2. .html File

  • Binary files are files that contain data in a format that is not human-readable. They can only be opened and edited using a program that understands the format
    Examples:

    1. .exe File
    2. .jpg File

3 -What is meant by lambda expression? Explain by real example

A lambda expression is a way to create a small anonymous function that can be passed as an argument to a method or stored in a variable. It allows you to write concise and expressive code that is more readable than using anonymous inner classes.
